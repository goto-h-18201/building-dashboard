# 建物エネルギー管理ダッシュボード

複数の建物を登録し、CO2排出量を一括で比較・分析するReactアプリです。

**デモ**: https://goto-h-18201.github.io/building-dashboard/

## 起動方法

```bash
npm install
npm run dev
```

---

## このアプリについて

建設業界ではカーボンニュートラルへの対応が急務です。
このアプリは、複数の建物のCO2排出量を一括で登録・比較し、
改修による削減ポテンシャルを可視化する管理ツールです。

**想定ユースケース**: 施設管理部門が複数建物の排出状況を把握し、改修の優先順位を判断する

**技術的なアピールポイント**:
- モダンなフレームワーク（React 19 + TypeScript + Vite 7）
- Tailwind CSS v4 によるユーティリティファーストのスタイリング
- Recharts によるデータ可視化
- 計算ロジック・状態管理・UIの明確な責務分離

## ファイル構成と設計の意図

```
src/
├── components/         # UIコンポーネント（1ファイル1コンポーネント）
│   ├── BuildingForm.tsx    # 建物登録フォーム
│   ├── BuildingList.tsx    # 登録建物一覧
│   ├── CO2Chart.tsx        # 比較棒グラフ
│   └── SummaryCards.tsx    # サマリーカード（合計CO2等）
├── hooks/
│   └── useBuildings.ts    # 建物データのCRUD + localStorage永続化
├── utils/
│   └── co2Calculator.ts   # CO2計算ロジック（純粋関数）
├── types/
│   └── building.ts        # 型定義
├── data/
│   └── sampleBuildings.ts # デモ用初期データ
├── App.tsx                # ルートコンポーネント
├── App.css                # Tailwind CSS v4 エントリポイント
└── main.tsx               # アプリのエントリポイント
```

### 設計方針

**計算ロジックの分離** (`utils/co2Calculator.ts`)
- CO2の計算式はコンポーネントに書かず、純粋関数として分離
- 係数の改定時はこのファイルだけ変更すればよい

**状態管理の集約** (`hooks/useBuildings.ts`)
- localStorage への読み書きをカスタムフックに集約
- コンポーネントは「建物の追加・削除」だけを呼び出す

## 計算ロジックの根拠

環境省・国土交通省の公開データに基づく。15パターンのテストケースで検証済み。

| 値 | 内容 | 出典 |
|----|------|------|
| 0.000423 tCO2/kWh | 電力CO2排出係数 | 環境省「電気事業者別排出係数」令和8年提出用 全国平均係数 |
| 用途別原単位 | 事務所220, 住宅130, 店舗330, 工場380 kWh/m²/年 | 国土交通省 (2009) をもとに算出 |
| 削減率 35% | 改修ポテンシャル | 国土交通省「建築物省エネ法」改修事例をもとにした目安値 |

## AIとの協働プロセス

### AIに任せた部分

- Reactコンポーネントのスケルトン生成
- Tailwind CSS v4 のクラス適用
- Recharts のグラフ設定

### 自分で判断・確認した部分

**アーキテクチャの決定**
計算ロジック（utils）、状態管理（hooks）、UI（components）の3層に分離する構成を指定。

**計算結果の整合性検証**
15パターンのテストケース（境界値・全用途・設備組合せ）で正確性を確認。

**技術選定の根拠**
Tailwind CSS v4 は v3 と大きく異なるため（設定ファイル不要、CSS import方式変更）、
最新のドキュメントを確認してから採用。
